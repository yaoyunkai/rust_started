# 闭包

普通函数只能使用通过参数传进来的数据，而闭包有一种“超能力”：它可以直接“捕获”并使用它周围作用域里的变量。

既然闭包可以捕获周围的变量，那么它是如何捕获的呢？根据我们之前讨论的 Rust 核心内存管理规则，无非只有三种方式：

1. 拿走所有权（`转移 Move`）
1. 可变借用（`&mut`）
1. 不可变借用（`&`）

为了在编译时保证内存安全，Rust 编译器会观察你的闭包内部是如何使用这些捕获变量的，并据此自动为闭包打上对应的标签（即实现对应的 Trait）。这就诞生了 `FnOnce`、`FnMut` 和 `Fn`。

- **`FnOnce`**：闭包内部拿走了捕获变量的“所有权”。因为数据被消耗掉了（比如被销毁或转移给了别人），所以这个闭包**只能被调用一次**（Once）。
- **`FnMut`**：闭包内部对捕获变量进行了“可变借用”。它可以修改外部环境变量。因为是可变借用，它可以被调用多次，但调用时闭包本身必须声明为可变的（`mut`）。
- **`Fn`**：闭包内部对捕获变量仅仅进行了“不可变借用”（或者完全没有捕获任何变量）。它只是读取数据，不修改、不消耗。它可以被调用无数次，甚至在多线程中同时被调用。
